# Security Policy

## Reporting a Vulnerability

BuildCycle handles cryptocurrency and smart contracts. Security is a top priority.

**Please do not file a public GitHub issue for suspected vulnerabilities.**

Instead, report directly to the core team by emailing [INSERT EMAIL].

You should receive a response within 48 hours. If you don't, follow up to ensure we received your report.

### What to include

- Description of the vulnerability
- Steps to reproduce (including any contract addresses, test accounts, or code snippets)
- Potential impact
- Suggested fix (if any)

## Scope

- Soroban smart contracts (`contracts/`)
- Backend API (`backend/`)
- Frontend wallet integrations (`frontend/`)
- Authentication and authorization flows
- Deployment and infrastructure configuration

## Out of Scope

- Issues in dependencies that are already reported upstream
- Theoretical attacks with no practical exploit path

## Process

1. You report the vulnerability via email
2. The core team acknowledges receipt within 48 hours
3. We investigate and develop a fix
4. A security advisory is published and a fix is released
5. You are credited for the discovery (unless you prefer to remain anonymous)

## Smart Contract Bug Bounty

A formal bug bounty program is planned for mainnet launch. For now, we deeply
appreciate responsible disclosure and will acknowledge all valid reports.
